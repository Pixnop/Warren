# Glossary

Shared vocabulary for Warren. Use these terms consistently in code, docs, and UI.

- Module: a placed instance of a module type (straight, elbow, tee, wye,
  transition, platform, cage_mount). Has parameters and a world transform. The
  unit a user adds to the scene. See [DATA_MODEL.md](./DATA_MODEL.md),
  [MODULES.md](./MODULES.md).

- Module type: the parametric definition (schema, ports, proxy, scad) that a
  module is an instance of. See [MODULES.md](./MODULES.md).

- Port: one connection interface on a module, modeled as an oriented frame
  (position + orientation) plus connector typing (gender and section). Local +Z
  points outward along the connection axis. See [DATA_MODEL.md](./DATA_MODEL.md),
  [CONNECTOR_SPEC.md](./CONNECTOR_SPEC.md).

- Open port: a port not currently part of any connection. Open ports are the
  snap targets in the editor and the only valid endpoints for a new connection.

- Connector: the single universal friction-fit standard that every port
  implements. Two genders, male and female; a male mates a female of the same
  section. See [CONNECTOR_SPEC.md](./CONNECTOR_SPEC.md).

- Gender: male or female. A connection joins one male port to one female port.

- Section: the connector cross-section, shape (round or square) plus nominal
  size (mm). Two ports mate only if their sections match. Round and square are
  not cross-compatible; use a transition module to bridge them.

- Keying / clocking / detrompage (poka-yoke): the anti-rotation feature that
  fixes a connection's rotational orientation, so asymmetric modules land in a
  known pose and the printed assembly matches the on-screen design. Round
  connectors use a key rib/notch; square connectors are keyed by their shape.

- Friction-fit: a press fit with no glue or screws, held by tuned clearance.
  Warren's connectors are friction-fit, tolerances tuned for the Bambu Lab P1S.

- Connection: an undirected mate between exactly two ports on two modules. See
  [DATA_MODEL.md](./DATA_MODEL.md).

- Graph: the project's source of truth, `Module[]` plus `Connection[]`. Both
  renderers are projections of it. See [ARCHITECTURE.md](./ARCHITECTURE.md).

- Proxy (proxy mesh): the cheap Three.js mesh used for fluid editing, distinct
  from the real OpenSCAD-generated mesh. See [ARCHITECTURE.md](./ARCHITECTURE.md).

- Real mesh: the printable geometry generated by openscad-wasm at preview/export
  time, validated by manifold-3d.

- Segment: a length of tube. Often a straight module, or the run portion of a
  tee/elbow. Informal; not a distinct data type.

- Bore: the continuous internal diameter (round) or inner side length (square)
  of a tube, kept continuous across a connector so the animal-facing surface is
  smooth.

- Print box: the usable print volume, mm, with an edge margin. Default models the
  Bambu Lab P1S (250 mm cube usable). Auto-split keeps every part within it. See
  [research/printing-and-materials.md](./research/printing-and-materials.md).

- Auto-split: the export step that cuts a part too large for the print box into
  sub-parts, inserting a connector pair at each cut. See [ROADMAP.md](./ROADMAP.md)
  Phase 5.

- Manifest (assembly manifest): the exported description of which printed parts
  connect to which, in what order, with which connector and clocking. Produced
  alongside the STLs. See [ARCHITECTURE.md](./ARCHITECTURE.md) Phase 6.

- Cage mount: the adjustable bracket module that clamps the network to the cage
  bars, sized from the cage's bar spacing and diameter. See
  [MODULES.md](./MODULES.md).

- Cage spec: the user's cage description (interior dimensions, bar spacing, bar
  diameter) in project settings. Drives the cage mount. See
  [DATA_MODEL.md](./DATA_MODEL.md).
