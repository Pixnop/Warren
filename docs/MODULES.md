# Module catalogue

Each module type is a parametric OpenSCAD definition plus a port layout. The
domain layer knows a module by its `type`, `parameters`, and the ports its
definition produces (see [DATA_MODEL.md](./DATA_MODEL.md)). The real printable
geometry is generated by openscad-wasm at preview/export time (see
[ARCHITECTURE.md](./ARCHITECTURE.md)).

Every port on every module is one half of the universal connector (see
[CONNECTOR_SPEC.md](./CONNECTOR_SPEC.md)). Parameter values below are starting
points to refine in Phase 4, not final.

## Shared parameters

Most modules accept these, defaulting from project settings unless overridden:

- `bore` (number, mm): internal diameter (round) or inner side (square). Must
  match the section of any port it connects to.
- `section` ('round' | 'square'): cross-section shape.
- `wall` (number, mm): wall thickness (default from `defaultWallThickness`).
- `vents` (boolean) and `ventPattern` (params): optional ventilation holes,
  teardrop-shaped so they print support-free.
- Per-port `gender` choices where a module's ports could be either, so users can
  resolve male/female along a run. Defaults chosen so a typical run alternates
  correctly.

## straight

A straight tube segment.

- Parameters: `bore`, `section`, `wall`, `length` (mm), `vents`.
- Ports: two, `a` and `b`, coaxial, at opposite ends, +Z pointing outward at
  each end. Opposite default genders so segments chain.
- Print note: printed vertically (axis along Z) for a support-free, smooth bore.
- Auto-split friendly: long runs are the most common reason for a split; a
  straight is trivially splittable at any plane along its length.

## elbow

A bend, default 90 degrees.

- Parameters: `bore`, `section`, `wall`, `angle` (deg, default 90),
  `bendRadius` (mm), `vents`.
- Ports: two, `a` and `b`, at the configured angle. Asymmetric, so a single-key
  (1-fold) round connector is used to fix orientation.
- Print note: orientation chosen so neither arm exceeds the support-free
  overhang limit; may print as an L resting on one arm. Validate in Phase 4.

## tee

A T junction, three ports.

- Parameters: `bore`, `section`, `wall`, `branchAngle` (deg, default 90),
  `vents`.
- Ports: three, `a` and `b` form the run (coaxial), `c` is the branch at
  `branchAngle`. Keyed so the branch direction is deterministic.

## wye

A Y junction, three ports at a shallow split.

- Parameters: `bore`, `section`, `wall`, `splitAngle` (deg, default 45),
  `vents`.
- Ports: three, `in` plus two branches at +/- `splitAngle/2`. Keyed.
- Note vs tee: gentler angle, easier for an animal to traverse.

## transition

Adapts between two different sections or sizes. The only way to bridge
round-to-square or two different nominal bores (see
[CONNECTOR_SPEC.md](./CONNECTOR_SPEC.md)).

- Parameters: `sectionA`, `boreA`, `sectionB`, `boreB`, `wall`, `length`.
- Ports: two, `a` (section/bore A) and `b` (section/bore B). The internal lofted
  transition keeps the bore smooth.

## platform

A flat resting area / chamber with one or more tube ports.

- Parameters: `width`, `depth`, `height`, `wall`, `floorVents` (boolean),
  `ports` (list describing each tube entry: position, section, bore, gender),
  edge `lip` height to stop bedding falling out.
- Ports: N, user-placed around the perimeter or on the floor/walls. Each is a
  standard connector. Keyed individually.
- Print note: large platforms are prime auto-split candidates; the split planner
  adds connectors at the cuts.

## cage_mount

The adjustable bracket that clamps the network to the cage bars. This is the
fixture that anchors everything to the user's actual cage.

- Parameters: `barSpacing` and `barDiameter` (default from `CageSpec`),
  `bars` (how many bars it spans), `grip` style, `wall`, plus one tube-side
  port.
- Ports: at least one standard connector facing the tube network; the cage side
  is not a connector but a clamp/hook sized to the bars.
- Adjustability: the clamp accommodates a range of bar spacings/diameters so it
  fits varied cages. Exact mechanism (hook, screw clamp, snap) to be designed in
  Phase 4 from the safety and printability rules.
- Print note: must hold weight without flexing loose; thicker walls / higher
  infill here.

## Module definition shape (for Phase 1 / Phase 4)

Each type is registered in a module library with:

```ts
interface ModuleDefinition<P> {
  type: ModuleType
  /** Validate and normalize raw parameters. */
  schema: ParamSchema<P>
  /** Port layout (ids, local frames, genders, sections) from parameters. */
  ports(params: P): Port[]
  /** Proxy geometry for the Three.js viewport (cheap). Phase 2. */
  proxy(params: P): ProxySpec
  /** OpenSCAD source (or builder) for the real mesh. Phase 4. */
  scad(params: P, connector: ConnectorSpec): string
}
```

Phase 1 needs only `schema` and `ports` (pure data, unit-testable). `proxy`
arrives in Phase 2, `scad` in Phase 4. See [ROADMAP.md](./ROADMAP.md).

## Open questions

- Default `bore` for the MVP (drives the single connector size). One size first.
- Which modules ship in the MVP vs later: `straight`, `elbow`, `tee`, and
  `cage_mount` are the minimum useful set; `wye`, `transition`, `platform` can
  follow.
- Whether platform ports are freely placed or snap to a coarse grid on the
  platform surface.
